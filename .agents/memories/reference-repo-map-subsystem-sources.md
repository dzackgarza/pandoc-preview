# Reference Repo Map: Subsystem Sources

**When this applies:** implementing any editor-shell subsystem (tree, tabs, panes, autosave, render lifecycle, math). Do not invent patterns — pull from these audited sources. Clean-room reference only for GPL/AGPL projects unless that license is explicitly accepted.

| Subsystem                                                                             | Source(s)                                                                                                                                                                      | Notes                                                                    |
| ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------ |
| Editor shell, split panes, recent files, file watching, drag/drop, status/dirty state | [Inkwell](https://github.com/Amoner/inkwell) (MIT)                                                                                                                             | Primary; vanilla JS + CodeMirror 6 + Tauri 2 dialog/fs/store plugins     |
| Project tree, workspace/folder tabs, sidebar, file-op UX, reveal-in-manager           | [Glyph](https://github.com/hamidfzm/glyph) (MIT), [marka.md](https://github.com/mattenarle10/markamd) (MIT), [mx](https://github.com/vibery-studio/mx) (GPL-3, reference only) | Backend scan via Rust, never a frontend path-entry picker                |
| File tabs, search/replace, autosave/crash recovery UX, version snapshots              | mx (GPL-3, reference only)                                                                                                                                                     | Best "what a shell should feel like" reference                           |
| Git autosave/recovery                                                                 | [SoloMD](https://github.com/zhitongblog/solomd) AutoGit concept; Glyph/mx `git2` (vendored libgit2) dependency pattern                                                         | Every save = real local commit; SoloMD itself too large to fork          |
| Pandoc subprocess + config                                                            | [mdTeX](https://github.com/slashinfty/mdTeX) (MIT)                                                                                                                             | Writes temp config/YAML then runs Pandoc with selected extensions/output |
| Backend compile/preview lifecycle (pending/success/failure, artifact reload)          | [Tideflow](https://github.com/BDenizKoca/Tideflow-md-to-pdf) (MIT)                                                                                                             | Typst/PDF there; target here is Pandoc stdout HTML, not a PDF artifact   |
| MathJax post-typeset after DOM injection                                              | MathJax official dynamic-typesetting docs; [Markdown Viewer](https://github.com/ThisIs-Developer/Markdown-Viewer), Gramax (GPL-3) as references                                | No KaTeX path anywhere                                                   |
| Native dialogs, menus, capabilities                                                   | Tauri v2 official docs (dialog plugin, menu API, capabilities)                                                                                                                 | Tauri owns the privileged surface; lock with capabilities                |
| Polished shell, autosave, command palette (reference only)                            | Astro Editor (AGPL)                                                                                                                                                            | Never code source                                                        |


**Decision context:** [Architecture Path: Fork-Biased, Inkwell First](architecture-path-fork-biased-inkwell-first). Theorem/callout/document semantics belong in the Pandoc template/filter asset layer, not in frontend Markdown interpretation.

**Verify:** any new shell subsystem PR can name its reference source from this table; no subsystem is invented from scratch when a row covers it.
